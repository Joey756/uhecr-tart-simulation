# UHECR Air Shower Radio Detection — CORSIKA/CoREAS Simulation

Investigating whether [TART](https://github.com/tart-telescope) (operating at 1.575 GHz) can detect radio emission from ultra-high energy cosmic ray (UHECR) air showers. This is a footprint demonstrator — the goal is to characterise the radio signal on the ground, not to estimate detection rates.

## Simulation Parameters

| Parameter | Value |
|---|---|
| Primary particle | Proton |
| Energy | 1 EeV (10¹⁸ eV) |
| Zenith angle | 45° |
| Azimuth | 0° (shower propagates north) |
| Hadronic model | SIBYLL + FLUKA |
| Mode | Thin (Wmax = 100) |
| Observation level | 1,400 m a.s.l. |
| Shower maximum (Xmax) | 670.7 g/cm² slant depth (~2,130 m from core) |
| Magnetic field (Otago, NZ) | B = (19.71, −14.18) μT |
| CORSIKA version | 7.8050 with CoREAS v1.4 |

## Antenna Grid

A 40 × 40 uniform grid spanning 500 m × 500 m centred on the shower core, at 1,400 m a.s.l. Antenna spacing is ~12.8 m. Positions are stored in `SIM000001.list` and generated by `generate_tart_config.py`.

CoREAS time resolution: 0.2 ns (Nyquist frequency ~2.5 GHz, covers the TART band at 1.575 GHz).

## Repository Structure

```
RUN000001.inp          # CORSIKA steering file
SIM000001.reas         # CoREAS parameter file
SIM000001.list         # Antenna positions (40×40 grid)
generate_tart_config.py  # Generates the three config files above
coreas_to_hdf5.py      # Converts CoREAS output to HDF5 for analysis
coreas_to_hdf5_README.txt
SIM000001_coreas/      # Raw electric field traces from first simulation run
all-inputs*            # Example CORSIKA input variants (Cherenkov, EPOS, NEXUS, etc.)
conex-*-inputs         # CONEX-mode input variants
parallel-inputs        # Template for parallel multi-core runs
tart_run_01.log        # Log from first simulation run
```

## Running the Simulation

**1. Generate configuration files**
```bash
python generate_tart_config.py
```
This writes `RUN000001.inp`, `SIM000001.reas`, and `SIM000001.list`.

**2. Run CORSIKA with CoREAS**
```bash
./corsika78050Linux_SIBYLL_fluka_thin_coreas < RUN000001.inp
```
Output goes to `SIM000001_coreas/` (one `.dat` file per antenna).

**3. Convert output to HDF5**
```bash
python coreas_to_hdf5.py SIM000001.reas
# Optional: compute energy fluence and other high-level quantities
python coreas_to_hdf5.py SIM000001.reas --highLevel
```
Requires [radiotools](https://github.com/nu-radio/radiotools):
```bash
git clone https://github.com/nu-radio/radiotools.git
export PYTHONPATH=$PYTHONPATH:/path/to/radiotools
```

## Dependencies

- CORSIKA 7.8050 (compiled with SIBYLL, FLUKA, thinning, CoREAS)
- Python 3 with: `numpy`, `scipy`, `h5py`
- [radiotools](https://github.com/nu-radio/radiotools) (for HDF5 conversion)

## Machine

Simulations run on `schmalzburg` (Tim Molteno's lab, University of Otago), single-core.
